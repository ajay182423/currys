1.    Process Overview

 

SRM Billing is responsible for preparing the statements for Owners and Club Members on account Maintenance Fee and Club Dues. These statements are prepared once the billing memorandums are received from the Resort Controllers or National Trust Director (Trust billings). These statements are mailed between August and October each year. Billing Processor are also responsible for running the late fee and interest assessments/ special assessments if required on Owner and Club accounts once past due to charge the accounts.

 

 

An Owner is a person who owns a deeded property with Sunterra.

 

A Developer is the party or corporate entity responsible for acquiring the land, desigining and building the resort, marketing and selling the weeks, and providing the service to the owners regarding reservation and financing.

 

 

 

 

 

2.    Annual Billing

 

Annual Billing is done during August to October with a Due Date of 1 January.

 

The data files for the billing are sent to the Printing Vendor for printing and upon approval of the proofs received, the vendor prints and mails the statements to the Club Members, Owners, and Developers etc for payments.

 

 

a) Owner and Developer Billing

 

The Resort Controllers prepare and approve the annual budget for their resort; the Budget Memorandum gives projected figures to be collected for the year. The Resort Controllers send these projected figures to Billing based on the following:

 

  • Unit Type
  • Number of Units
  • Maintenance Fee
  • Reserve Fee
  • Total Billing for the resort

 

Follow the steps below to complete annual billing:

 

i.     Setting up Assessment Tables

 

Setting up the assessment table on Atlas will allow Atlas to bill as per the assessment criteria’s entered. Refer to the Billing Memo to input these details.

 

  • Click on ‘Owners’ tab in ‘Sales and Marketing’ region
  • Click on ‘Owners Receivables’
  • Click on ‘Receivables Setup’
  • Click on ‘Assessment Tables’
  • Click on ‘Add’ tab to add the parameters to setup the assessment tables
  • Enter a 3-letter table code to identify the table. Type YYYY (upcoming billing year)+ Inv Mfee and additional levies (As per Budget Memo) in the Description field. Select ‘Yes’ in the ‘Active’ field. Select ‘No’ in ‘Special Assessment’ field. Select ‘Billing Currency’ as ‘USD-US DOLLARS’.
  • Upon clicking ‘Save’, the new 3-letter code will be saved at the bottom in the Code field. Click the 3-letter code to be taken to the Assessment Tables Change screen.
  • Select the ‘Price Type’, ‘Calc Method’ and ‘Transaction Code’ as per the billing memo. Key in the ‘Flat Amt’ as ‘0.00’.
  • Click ‘Back’

 

 

Note1: The Processor will add 3.00 (ARDA Fee) in the Flat Amt field if processing UDI billing.

 

Note 2: The Processor will follow the same steps for setting up tables for Developer billing. Example: 2005 Dev Unit type w/Taxes.

 

 

ii.    Assessment Pricing

 

Follow the below mentioned steps to enter fees applicable for the resort billing as per the Billing Memo.

 

Assessment Pricing for Deeded Week:

  • Click on ‘Assessment Pricing’ in the ‘Owners Receivables Setup Menu’
  • Select the appropriate resort in the ‘Project’ field and select the upcoming year in the ‘Assessment Year’ field
  • Click ‘Process’
  • Click the ‘Add’ tab to enter the pricing amounts as per the Resort Billing Memo
  • Select the appropriate ‘Calculation Method’ and click ‘Process’
  • Select the appropriate ‘Unit Type’ as per Billing Memo and click ‘Process’
  • Add all the ‘Unit Type’ as per the billing memo
  • Click on the hotlink of each Unit Type added, to add the pricing
  • Select the ‘Assessment Type’ as the billing memo. Select ‘Odd/Even/Both’ as per the Assessment Parameters List and enter the ‘Flat Price’ as the billing memo.
  • Click ‘Back’ button all the way to go to Owners Receivables Setup Menu

 

 

Assessment Pricing for UDI Property:

  • Select ‘Calculation Method’ as ‘Per Point’ and Click ‘Process’
  • Select the appropriate method from the ‘Points Group’ field and click ‘Process’
  • Select the ‘Assessment Type’ and ‘Price/Point’ as per the billing memo and click ‘Save’
  • Once all the assessment type have been added, click on ‘back’ button all the way to go to ‘Owners Receivables Setup Menu’
  • Click on ‘Project Assignment’
  • Select the Resort as the ‘Project Assignment’
  • Enter ‘Inventory Assess Plan’, ‘UDI Assess Plan’, ‘Developer Inventory Assess Plan’, Developer Assess Plan’ and ‘Trust Assess Plan’ as per the assessment tables created.
  • Click ‘Save’
  • Click the ‘Back’ button to return to Owners Receivables Setup Menu

 

 

 

 

iii.   Running Projected Budget

 

  • Go to ‘Owners Receivables Main Manu’
  • Click on ‘Reports’
  • Click on ‘Assessment Budget’
  • Select the ‘Project’ as per the Billing Memo, ‘Assessment Year’ as the upcoming year and ‘Detail Report’ as ‘No’
  • Click ‘Process’
  • The Project Assessment Budget Report will populate in an excel format
  • Match the report with the Memorandum

 

 

iv.   Running the Assessment

 

Deeded and UDI Assessments
  • Click on ‘Assessment’ Tab in Owners Receivables menu
  • Select ‘Invoice Date’ and ‘Due Date’ as first day of the year.
  • Select ‘Currency’ as ‘USD-US DOLLARS’.
  • Select the appropriate ‘UDI or Inventory?’ type.
  • Select ‘Table’ as created for the resort.
  • Select ‘Project’ as the resort for which assessment has to be run.
  • Select ‘Owners Flag’ as ‘All’.
  • Select ‘Excluded Owners Flags’ as ‘All’ except in for the following resorts.
  • Flamingo Beach I – Select Lock off Owners (LOCL)
  • Royal Palm Beach- Select Home Owners (RWOR)
  • Villas of Santa Fe- Select SUN9
  • Click on ‘Back’ button
  • Click on ‘Transaction Entry’
  • Click on ‘Print Edit List’
  • Click on the ‘Batch Id’ hotlink
  • Copy of the Excel file generated will be emailed to the resort controller for approval

 

 

Developer and Trusts Assessments
  • Click on ‘Owners’ tab in ‘Sales and Marketing’ field
  • Click on ‘Owner Receivables’
  • Click on ‘Developer Functions’
  • Click on ‘Developer Assessment’
  • Select the ‘Inventory Project’ as the resort for which the assessment is to be run
  • Select the ‘Assessment Year’ as the upcoming year
  • Select ‘Invoice Date’ and ‘Due Date’ as first day of the year
  • Select the appropriate ‘Inventory, UDI or Trust?’ type.
  • Click ‘Process’
  • Click on ‘Back’ button
  • Click on ‘Print Edit List’
  • Click on the ‘Batch Id’ hotlink
  • Copy of the Excel file generated will be emailed to the resort controller for approval

 

 

 

 

v.    Posting Transaction

  • Click on ‘Post Transactions’
  • Click on the ‘Batch Id’ hotlink
  • Click ‘Process’
  • Click on ‘Print Journal’ and save the file on the shared drive

 

 

 

 

vi.   Creating the Statement File

  • Click on ‘Reports’ under ‘Owner Receivables’ menu
  • Click on ‘Statements to File’
  • Select ‘Owner Flag’ and ‘Exclude Selected Owner Flags’ to move to ‘Parameters Selected’ region
  • Enter the ‘Transaction Date’ and the ‘Past Due as of Date’ as the first date of the upcoming year
  • Select ‘Include Credit Balance’ as ‘Yes’. Select as ‘No’ in case the credit balances and zero balances need to be excluded on the billing statement
  • Select Project as per Billing Memo
  • Select ‘Exclude Selected Owner Flags’ as ‘Yes’
  • Select the ‘Owner Flags’ that need to be excluded
  • Select ‘Starting Last Name Character From’ as ‘A’
  • Select ‘Starting Last Name Character To’ as ‘F’
  • Go through the data files to verify Owner information, True balance and that the current years fees are billed
  • Save the file on the shared drive

2.    Annual Delinquency Billing

 

The owners/members are mailed the ABC Policy (Assessment Billing and Collections) with their billing statements each year. The ABC Policy details the late fee, time frame and due dates within which the HOA Fees, Club Dues or Maintenance Fee payments can be made. We start assessing the delinquency billings from January 5th of each year as the due date for paying the fees according to resorts is generally January, 1st. Annual Late Fees are charged only once a year and the interest is charged every  month.

 

a) Annual Owner Delinquency Billing

 

Follow the steps given below to complete the delinquency billing for owner accounts:

  • Click on ‘Owners’ tab in ‘Sales and Marketing’ field
  • Click on ‘Owner Receivables’
  • Click on ‘Receivables Setup’
  • Click on ‘Delinquency Methods’
  • Create a 3 digit code in ‘Delinquency Method Code’ field
  • Enter ‘<YYYY> <Resort Code> Late Fee’ in the ‘Description field
  • Enter the ‘Flat Amount’ of the late fee as per the ABC Policy.
  • Enter ‘Minimum Balance’ as ‘100’ unless objected by resort controllers
  • Enter ‘Minimum Days Past Due’ as per ABC Policy
  • Click ‘Save’
  • Click the ‘Back’ button all the way to be taken to ‘Owners Receivables Setup Menu’
  • Click on ‘Project Assignments’
  • Select the ‘Project Id’ and click ‘Search’
  • Select the ‘Delinquency method for late fees’ as per the ABC Policy
  • Select the ‘Delinquency method for Fin charge’ as per the ABC Policy
  • Click ‘Save’
  • Click the ‘Back’ button all the way to be taken to ‘Owners Receivables Setup Menu’
  • Click on ‘Owners Receivables Main’ hotlink
  • Click on ‘Delinquency Assessment’
  • Select ‘Type’ as ‘Late Fee’
  • Enter the current date in the ‘Balance Date’ field
  • Enter the ‘Invoice Date’ as per the ABC policy for the resort
  • Enter the ‘Due Date’ as per the ABC policy for the resort
  • Select the ‘Transaction Code’ created
  • Select the following ‘Apply Codes’:
    • City Tax
    • Replacement Reserve
    • Real Estate Tax
    • Maintenance Fee
    • Base Flex Ownership
  • Select the appropriate ‘Project’
  • Select ‘All’ in ‘Owner Flag Include’ field
  • Select all Bankruptcy, Do not Assess, Escalated Complaints, Disputed Flags, etc in the ‘Owner Flags Exclude’ field
  • Click the ‘Back’ button all the way to be taken to ‘Owners Receivables Main Menu’
  • Click on ‘Transaction Entry’
  • Click on ‘Print Edit List’
  • Click on the ‘Batch Id’ hotlink
  • Verify fee on Atlas for few accounts
  • Save the file on the shared drive

 

 

Posting Batch:

Once the approval is received on the file generated, the payments will be posted on Atlas.

 

  • Click on ‘Transaction Entry’
  • Click on ‘Post Transactions’
  • Click on the ‘Batch Id’ hotlink
  • Click on ‘Process’
  • Click on ‘Print Journal’ and save it on the shared drive
  • Click on the ‘Back’ button to go all the way to ‘Owner Receivables Menu’

 

 

Printing Statements:

  • Click on ‘Reports’ tab
  • Click on ‘Statements to File’
  • Select ‘Owner Flag’ and ‘Exclude Selected Owner Flag’ and move it to ‘Parameters Selected’ field
  • Click ‘OK’
  • Select the first day of the current year in the ‘Transaction Date To’ field
  • Select the first day of the current year in the ‘Past Due as of Date’ field
  • Select ‘Yes’ in ‘Include credit balances’ field
  • Select ‘Project’ as per billing memo field
  • Select ‘Yes’ in ‘Exclude Selected Owner Flags’ field
  • Selected all Bankruptcy and Do Not Use codes in ‘Owner Flags’ field
  • Select ‘A’ in ‘Starting Last Name Character From’ field
  • Select ‘F’ in ‘Starting Last Name Character To’ field
  • Click ‘Process’
  • Save the file on the shared drive

5.    Apply Open Credits

 

Any non-applied payments in the account need to be applied to the oldest invoice outstanding. This report is run on the last business day of the month.

 

Accounts with a negative dollar amount showing as due, are accounts that have credits. When the account is invoiced this credit does not automatically go to the outstanding invoice. In order to apply the funds correctly, this procedure must be completed. This procedure removes credits and applies funds to invoiced fees.

 

  • Enter the appropriate Club/Owner Module
  • Click on ‘Owner/Club Receivables’
  • Click on ‘Transaction Entry’
  • Click on ‘Apply Open Credits’
  • Enter the ‘Transaction Date’ as the current date
  • Select the ‘Unapplied Credit Code’ as ‘xPrepaid Credit System Use’
  • Select ‘Debit Memo Code’ as ‘Transfer to Open Credit’
  • Select ‘Credit Memo code’ as ‘Transfer from Open Credit’
  • Select appropriate resort as the ‘Project’
  • Enter ‘Document Number’ as ‘1’
  • Click ‘Process’
  • Click the ‘Back’ button all the way to ‘Owner/Club Transaction Entry’ menu
  • Click on ‘Print Edit List’
  • Click on the ‘Batch Id’ hot link
  • Save the Excel file on the shared drive

 

Posting Transactions:

 

Once the file is approved the payments need to be posted on the accounts.

 

  • Click on ‘Post Transactions’ under the Owner/Club Transaction Entry Menu
  • Click on the ‘Batch Id’ hotlink
  • Click ‘Process’
  • Click on ‘Print Journal’ and save it on the shared drive
